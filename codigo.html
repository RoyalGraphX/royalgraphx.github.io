Apple's Subdomain Loophole.

This Exploit was found on November 3rd, 2019 and it came from nowhere. As a person usually interested in backdoors and other secrets, I went on the search for a possible Apple Store key generator, which would allow me to create many keys at a time, and valid for redeeming. In the beginning, I was lead to believe that it was special software at work, generating these based on numerical expressions, that once validated could in turn lead to successful redemption of these codes. When searching for what I wanted to actually get, I instantly wondered why I still was a monthy subscriber of Apple Music.

OpenBullet is a webtesting suite that allows to perform requests towards a target webapp and offers a lot of tools to work with the results. This software can be used for scraping and parsing data, automated pentesting, unit testing through selenium and much more, I was shared an existing .LOLI file for this software. Let's view it's contents.

When opening a Terminal window and closer examining the .LOLI file provided to be by a source I cannot comment on, it is clear that we are dealing with a script meant to scrape the iTunes Store for codes. Easier said than done though. Let's break down this file closer and preform some basic reverse engineering for deeper understanding at how this is meant to work.

This gives us some basic information like the configurations name, suggested bot #, It's author, Version #, and a few other instructions meant to be read by OpenBullet, Now as we scroll further, that's really where you begin to realize, This is hosted live on HTTPS! Anyone can access this website without any credentials. This was huge!

Firstly, the script will utilize it's built in ReCaptcha bypass, by sending a request to it's respective API and mask the User-Agent to the following HEADER information. After this, we go right into the direct website to retrieve a key.

But this is where it gets serious, If you could see this screenshot above without any of the censorship, you'd have the exploit itself, we directly send a REQUEST to the HTTP host, in this case, it's actually Apple's own website.

Turns out, they have a built-in API for generating codes for Apple Music, these are voucher codes for 1 month, and if you've never used your free trial, you get a whole 4 months of Apple Music for free, but what's the catch? Simple.

These code's are generated from an italian Apple server, a subdomain of the iTunes Store Redeem API. Simply changing your accounts region to Italy mitigates this issue and allows you to redeem these codes. That's it!
