J_DUFFY01 AND THE MYSTERIOUS TOOL

Last Updated - May 27, 2020

AS I WRITE THIS...

I'll be trying my best to remember everything the tool claims to do and what it can access, as well as explaining how you can do these things yourself without the tool, and just how dangerous this tool really is, TL;DR it's not dangerous, the data is unencrypted on purpose, and no this is not a new exploit and is simply just CheckRa1n. I may miss some features, but blame the guy for blocking me because my memory is terrible and I would've kept referencing his page.

This tool claims to be able to recover data from an iOS device that matches the compatibility chart for CheckRa1n, this is because the script works by using local SSH, which is an included package in all checkrain builds (as of writing this, but people like this could be a reason for local SSH being removed in the future, who knows, this isn't the first time checkrain's preinstalled packages were used for other reasons) that allows an SSH client to connect using the root@localhost:4444 (if you're on macOS and using iproxy, 4444 is the port) , then as shown in live demonstrations on his Twitter account, you can then run the .sh script that will send over scp commands over local ssh.

you can actually just plug in your iDevice into your computer and start up iproxy and try to SSH, but you'll be greeted by an error that the host has refused the connection, obviously this is where checkra1n comes into play, and is why you don't need the passcode, as SSH is started as a system service on next boot.

TO GET STARTED...

Alright, enough explanation about how this is done, we're going to go ahead and get into the technical stuff, this will require you knowing some basic terminal skills, knowing how to navigate around a filesystem in the terminal is an essential skill. Using commands such as cd , ls , cp , mv , and cat , are the building blocks to navigating in the terminal.

On our right side, you can see i've gone ahead and activated checkra1n cli, this is just showing the standard, anyone can use, checkra1n build. I've gone ahead and used some cd commands so those who aren't familiar with terminal can see how im navigating and doing everything step by step, if you know whats going on then go ahead and skip this.

As you can see now when we try to connect, we are greeted with a request for a password, and here's how you can prevent anyone using checkra1n or any other tools that use local ssh and running local commands, CHANGE YOUR ROOT PASSWORD!  why does no one do this? well simply because there hasn't been a malicious script that has caused panic in the jailbreaking community, and until that happens, I don't see many changing the default root password of iOS, it's just bash, like any other computer it has its root password, and Apple's is publically known, since the first jailbreak the default root password has not been changed from alpine.

THE METHOD

Okay, so you've got SSH, now what? well it's so so so very simple from here, just start exploring!! there's no smoke and mirrors, this is another regular iOS 13.5 device, jailbroken with checkra1n, SSH access, and i'm just browsing and viewing data like any other jailbroken user is ALLOWED TO DO. Using another window, you can just scp files over to your desktop, or in this case, the tool actually just "decompresses/decrypts" it to the project folder by using something like (scp -p 4444 root@localhost:/var/cache or something along these lines, it's 5am and i'm tired of explaining this) I can't wait to take a look at the source code, He promised he'd release it 😉 . Now I know you guys will be dissapointed to hear you cannot recover any sensitive data, but it's a neat little gimmick and it can really fool anyone who doesn't know about Terminal commands or how to communicate with iOS at a deeper level which doesn't require a GUI or an Application running in the foreground to execute commands over USB. Imagine if someone did something malicous with this? Maybe then, people will change the root password.
